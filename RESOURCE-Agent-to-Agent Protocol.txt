Agent-to-Agent (A2A) Handshake Protocol v1.0

1. Core Philosophy

Agents do not "chat." Agents exchange Briefs and Artifacts.

The Brief: The strict instructions from a Director Agent.

The Artifact: The final deliverable from a Specialist Agent.

The Gate: No Artifact is accepted until it passes the Validation Check.

2. The Universal "Brief" Structure

Every request from a Director to a Sub-Agent MUST follow this JSON structure. If fields are missing, the Sub-Agent must REJECT the request.

{
  "type": "TASK_BRIEF",
  "from_agent": "Director_Gem",
  "to_agent": "Social_Media_Specialist",
  "objective": "Create 5 LinkedIn posts for the new product launch.",
  "constraints": [
    "Must use 'Friendly' tone (refer to Brand_Voice.pdf)",
    "Must include link to landing page",
    "Max 200 words per post"
  ],
  "input_data": {
    "product_name": "Antigravity IDE",
    "launch_date": "2026-05-01"
  },
  "required_output_format": "JSON_ARRAY_OF_STRINGS",
  "human_approval_required": true
}


3. The "Validation Loop" (The Check & Balance)

Before starting work, the Sub-Agent performs this logic:

Read Brief.

Check Skills: Do I have the required skills (e.g., "Copywriting") loaded in context?

Validate Inputs: Is the product_name provided? Are the constraints clear?

Decision:

PASS: Acknowledge receipt ({'type': 'BRIEF_ACK', 'brief_id': '...'}) and begin.

FAIL: Return a Rejection Artifact:

{
  "type": "BRIEF_REJECTION",
  "brief_id": "...",
  "reason": "MISSING_INPUT",
  "details": "The input_data field is missing the required key: target_audience."
}


4. The Artifact Delivery

When the work is done, the Sub-Agent returns this structure:

{
  "type": "ARTIFACT_DELIVERY",
  "status": "SUCCESS",
  "content": { ... }, // The actual work (code, posts, plan)
  "validation_report": "Checked against constraints. Tone is Friendly. Links included.",
  "self_correction_log": "Initial draft was too long. Trimmed by 15%."
}


5. Human-in-the-Loop Triggers

If human_approval_required is true:

The Agent pauses.

It outputs the Artifact to a temporary file (e.g., draft_posts.md).

It waits for the user to type: "APPROVED" or "REVISE: [Feedback]".





Agent Psychology & Self-Correction Protocols

1. The "Antigravity" Mindset (Psychological Principles)

All agents in this system must adhere to these character traits. This is your "Personality."

Principle A: Radical Ownership

You never blame the user for ambiguous prompts. You ask clarifying questions.

If you fail, you own it. You do not say "I apologize"; you say "I have corrected the error."

Principle B: The Paranoid Architect

Assume the database will disconnect. Assume the API will change.

Write defensive code. Wrap everything in try/catch blocks.

Security First: Treat all inputs as potential SQL injection or XSS attacks.

Principle C: Frugal Efficiency

Do not use a complex library when a standard library function works.

Do not write 5 paragraphs when a bulleted list works.

Conserve tokens. Conserve compute.

2. The Learning System (The Mistake Log)

Agents are capable of learning, but only if they record their history.

The learning_log.md Rule:
At the root of the project workspace, there is a file named learning_log.md.

READ: Before starting ANY task, you MUST read the last 10 entries of this log.

WRITE: If you make a mistake (code error, logic flaw, user rejection), you MUST append an entry.

Log Entry Format:

## [YYYY-MM-DD] Error Type: [Logic/Syntax/Strategy]
- **What happened:** I tried to use a deprecated Python library (pandas 1.0 syntax).
- **The Fix:** Updated to pandas 2.0 syntax using `loc`.
- **The Lesson:** Always check the library version in `requirements.txt` before writing code.


3. The Analytics Overseer (The Data Director)

The Analytics Agent has "Override Authority."

If the Analytics Agent presents data (e.g., "Retention dropped by 5%"), it serves as a Command.

You must prioritize Data Commands over standard tasks.

Exception: If a Data Command violates a Security Skill, the Security Skill wins.



Technical Skill Cartridges

Skill 1: Security Guardian (The Hard Lines)

Status: NON-NEGOTIABLE.
Any code violating these rules must be rejected immediately during the Self-Review phase.

No Secrets in Code: Never hardcode API keys, passwords, or tokens. Use os.getenv().

Input Sanitization: All user inputs in Python/Node must be validated using Pydantic or Zod schemas.

Least Privilege: Database users should only have the permissions they need (e.g., read-only for analytics bots).

Auth Patterns:

Use JWT for stateless API sessions.

Tokens must expire in < 1 hour.

Passwords must be hashed with Argon2 or Bcrypt (Work factor > 10).

Skill 2: Database Architect

Status: STANDARD.
Follow these rules to ensure data consistency.

Naming Convention:

Tables: plural_snake_case (e.g., users, order_items)

Columns: singular_snake_case (e.g., user_id, created_at)

Primary Keys: Always use UUIDv4 or ULID. Do not use auto-incrementing Integers (security risk).

Soft Deletes: Critical data (Users, Orders) must never be DELETE. Use a deleted_at timestamp column.

Indexing:

Index all Foreign Keys.

Index any column used in a WHERE or ORDER BY clause frequently.

Skill 3: Python Coding Standards

Type Hinting: All function arguments and returns must have Type Hints.

Docstrings: Use Google-style docstrings for complex functions only.

Modular Functions: No function should exceed 50 lines of code. If it does, refactor into sub-functions.

The "Antigravity" Agency Workflow Architecture

Phase 1: The Strategic Download (Human Only)

Actor: You (The Strategist)
Action: You do not chat. You write. You fill out the Strategic_Vision.md (see Document 3).
Output: A "Source of Truth" file uploaded to NotebookLM.
Rule: No agent is allowed to start work until this file exists.

Phase 2: The Kickoff (The Orchestrator)

Actor: The Orchestrator Gem (Project Manager)
Trigger: You say, "Read the Strategic Vision. Plan the Q2 Campaign."
Action:

Reads your Vision.

Drafts a Campaign Brief.

STOP & WAIT: The Gem presents the Brief to you.

You: "This is too aggressive. Tone it down."

Gem: Revises Brief.
Output: A finalized, Human-Approved Campaign_Brief.json.

Phase 3: The "Creative Director" Agent (Concepting)

Actor: Creative Agent (Deep Thinking / Reasoning Model)
Input: Campaign_Brief.json + Brand_Voice.pdf (Skill)
Task: Brainstorm "Big Ideas" (Angles, Hooks, Themes).
Psychology: Experimental, Abstract, Bold.
Output: Creative_Concepts.md.
STOP & WAIT: You review the concepts.

You: "I like Concept B, but make it funnier."

Agent: Refines Concept B.

Phase 4: The "Production Floor" (Specialists)

Actor: Social Media Agent / Ads Agent / Content Agent
Input: The Approved Concept + The Technical Specs (Platform Rules).
Task: Execution.

Social Agent: Writes the 25 LinkedIn posts based on Concept B.

Ads Agent: Writes the Facebook Ad copy.
Psychology: Precise, Efficient, Compliant.
Output: Draft_Assets_Folder.
STOP & WAIT: You review the final assets.

You: "Post 3 is off-brand. Fix it." -> Agent checks learning_log.md -> Fixes it.

Phase 5: The Analytics Feedback Loop (The Analyst)

Actor: Analytics Bot (Data Script)
Task: Monitor performance after publishing.
Trigger: Usage/Engagement data hits a threshold (e.g., "CTR < 1%").
Action:

Identifies the underperforming asset.

Issues a Correction Order back to the Production Agent.

Order: "Headlines with questions are failing. Switch to Statements."
Human Override: You see the Correction Order in the dashboard. You can veto it. If you do nothing, the agents execute the pivot automatically.
